# Adaptive Pace 소셜 자동화 — MVP 구현 레퍼런스

> 6주 번아웃 감지 → Active↔Survival Mode 자동 전환 소셜 자동화 도구

---

## 1. 제품 개요

크리에이터/스몰 비즈니스 오너가 소셜 미디어 콘텐츠를 예약 발송하다가 번아웃에 빠지면,
시스템이 자동으로 감지하고 과거 고성과 콘텐츠를 재가공하여 계속 발행해주는 도구.

**핵심 메시지**: "시스템이 안전하게 유지 중입니다. 쉬고 오세요."

---

## 2. 타겟

- 매주 12개 이상 게시물을 올리는 스몰 비즈니스 오너 / 1인 크리에이터
- Buffer($6/월)~MeetEdgar($30/월) 가격대에서 소셜 스케줄링 도구를 이미 사용 중인 사람
- 번아웃 경험률 41~62% (MBO Partners 2024, Billion Dollar Boy, Creators 4 Mental Health)
- 주 9시간 이상 소셜 미디어 관리에 투자하는 크리에이터

---

## 3. 순환 메커니즘

**Active Mode (열정적 생산) ↔ Survival Mode (시스템 자동 인계)**

```
번아웃 → 복귀 → 번아웃 = 6주 주기 내부 순환
```

- **콘텐츠 소비 순환 (메커니즘 #1)**: 에버그린 콘텐츠 재가공 → 소비 완료 → 새 배치 필요
- **트렌드/유행 순환 (메커니즘 #2)**: Active↔Survival 모드 전환이 번아웃 주기에 맞춰 반복
- **스킬 진행 (메커니즘 #5)**: 복귀할 때마다 새 Active 사이클 시작

OneUpApp이 "콘텐츠 자동 재활용"만으로 $83K~$100K MRR 달성 = 이 순환의 WTP 검증됨.

## 5. 기술 아키텍처

### 5.1 Active Mode (사용자 활성 상태)

```
사용자가 콘텐츠 큐에 채움
  ↓
다중 소셜 API 예약 발송 (Threads, Bluesky, LinkedIn)
  ↓
발행 후 인게이지먼트 데이터 수집 (좋아요, 리포스트, 댓글)
  ↓
콘텐츠별 성과 점수 계산 + 저장
```

### 5.2 Survival Mode (번아웃 감지 시)

```
7일+ 큐 비어있음 감지 (포스팅 빈도 하락)
  ↓
번아웃 자동 판별 (규칙 기반, MVP)
  ↓
과거 인게이지먼트 상위 10% 콘텐츠 추출
  ↓
LLM 트렌드 맞춤 변형 (형식 변환: 트윗→Threads, 블로그→LinkedIn)
  ↓
주 2회 자동 발행
  ↓
"시스템이 안전하게 유지 중입니다. 쉬고 오세요" 알림
```

### 5.3 번아웃 감지 로직 (MVP: 규칙 기반)

```
IF 큐 비어있음 >= 7일
  AND 최근 14일 포스팅 빈도 < 직전 30일 평균의 30%
  AND 수동 Pause 미활성
THEN → Survival Mode 자동 전환
```

- MVP에서는 규칙 기반으로 시작
- ML 번아웃 예측(포스팅 빈도 변화, engagement 패턴, 콘텐츠 품질 지표)은 데이터 축적 후 Phase 2에서 추가

### 5.4 에버그린 콘텐츠 재가공 엔진

1. **콘텐츠 점수 매기기**: 인게이지먼트(좋아요, 리포스트, 댓글) 기반 상위 10% 자동 추출
2. **형식 변환**: 원본 플랫폼 → 타겟 플랫폼 포맷 자동 변환
   - 트윗 → Threads 캐러셀
   - 블로그 요약 → LinkedIn 포스트
   - 긴 포스트 → Bluesky 쓰레드
3. **트렌드 맞춤**: LLM으로 현재 트렌드/해시태그 반영하여 변형 (복사가 아닌 재가공)
4. **중복 방지**: 최근 30일 내 재발행된 콘텐츠 제외, 최소 재발행 간격 설정

### 해자: "역인센티브 니치"

VC 기업(Buffer, Hootsuite, Sprout Social)의 핵심 KPI = MAU, 사용 시간, 포스트 수.
**"사용자에게 쉬라고 하는 기능"은 VC 메트릭과 정면 충돌.**

- Buffer가 Pause를 "mental health toggle"이라 부르지만 적극 push 안 하는 이유
- VC가 구조적으로 들어오기 어려운 특수 니치 = 인디 개발자의 기회
- OneUpApp이 이 "역인센티브 니치"에서 2인 팀으로 $83K~$100K MRR 달성

## 10. MVP 로드맵

### Phase 0: 기반 구축 (2주)

- [ ] Next.js + Supabase 프로젝트 세팅
- [ ] Supabase Auth + 소셜 OAuth 연동
- [ ] LemonSqueezy 결제 연동 ($15/월 Pro 플랜)
- [ ] 기본 UI 레이아웃 (대시보드, 큐 관리, 설정)

### Phase 1: 핵심 기능 (4~6주)

- [ ] **소셜 API 연동**: Threads, Bluesky, LinkedIn 포스팅
- [ ] **콘텐츠 큐**: 작성 → 예약 → 발행 워크플로우
- [ ] **인게이지먼트 추적**: 발행 후 성과 데이터 수집 + 점수화
- [ ] **Survival Mode**: 큐 비어있음 7일+ 감지 → 에버그린 콘텐츠 자동 재가공 → 주 2회 발행
- [ ] **번아웃 감지 (규칙 기반)**: 포스팅 빈도 하락 감지 → 모드 자동 전환
- [ ] **알림 시스템**: "시스템이 안전하게 유지 중입니다" 이메일/인앱 알림
- [ ] **ProductHunt 론칭 준비**

### Phase 2: 리텐션 강화 (론칭 후 2~3개월)

- [ ] 번아웃 감지 정확도 개선 (데이터 기반 규칙 튜닝)
- [ ] 에버그린 재가공 품질 향상 (LLM 프롬프트 최적화)
- [ ] 사용자 피드백 기반 UX 개선
- [ ] 리텐션 메트릭 측정 (월간 리텐션, 해지율)

### Phase 3: 스케일 (MRR $500+ 후)

- [ ] X API 추가 ($100/월 정당화 가능한 시점)
- [ ] ML 기반 번아웃 예측 (충분한 데이터 축적 후)
- [ ] Team 플랜 추가 ($29/월)
- [ ] 분석 대시보드 고도화
