# 글로벌 SaaS 구독 결제 인프라 및 한국 세무 규제 심층 분석 보고서

## 1. 크로스보더 마이크로 SaaS 및 결제 인프라의 구조적 특성

현대 디지털 경제에서 소프트웨어 서비스(SaaS) 산업은 국경의 제약을 받지 않는 대표적인 크로스보더(Cross-border) 비즈니스 모델로 자리 잡았다. 특히 한국에 거주하는 개인 개발자가 사업자 등록과 같은 행정적 절차를 보류한 상태에서 영어권 유저를 타겟으로 월간 반복 매출(Monthly Recurring Revenue, 이하 MRR)을 창출하는 구독형 웹 기반 애플리케이션을 구축하는 사례가 급증하고 있다.

결제 인프라 산업은 규제 부담을 누가 지느냐에 따라 크게 두 가지 축으로 나뉜다:

- **PSP (Payment Service Provider)**: 단순히 자금의 이동 경로와 결제 게이트웨이만을 제공. 글로벌 세금 징수와 송금에 대한 법적 책임은 판매자 본인에게 귀속.
- **MoR (Merchant of Record)**: 법적으로 플랫폼이 개발자의 소프트웨어를 도매로 매입하여 최종 소비자에게 재판매하는 구조. 세금 계산, 징수, 납부 및 환불 처리에 대한 모든 법적 책임을 플랫폼이 전담.

## 2. 결제 플랫폼별 실사 및 심층 분석

### 2.1 LemonSqueezy (MoR)

- **사업자 없이 가입 가능**: 비즈니스 유형을 '개인(Individual)'으로 설정하여 즉시 상점 활성화
- **KYC**: 한국 여권(MRZ 스캔) 권장. 영문 운전면허증도 가능하나 여권이 승인율 높음
- **수익 수취**: SWIFT 은행 이체 또는 PayPal. Wise 연동도 가능 (명의 일치 필수)
- **세금 처리**: 미국 주별 판매세, EU VAT, 호주 GST 등 자동 계산·징수·납부 대행
- **수수료**: 5% + $0.50/건. 월 고정비 없음
- **송금 간접 비용**: 크로스보더 송금 수수료 + 환전 마진 약 1.5~2%. 한국 수취 은행 타발 송금 수수료 약 10,000원
- **출금**: 최소 $50, 월 2회 (15일/말일) 또는 Net-15

### 2.2 Gumroad (MoR)

- **사업자 없이 가입 가능**: 이메일만으로 즉시 계정 생성, KYC 심사 없음
- **월 구독 지원**: 기본적으로 지원하나 SaaS 자동화(라이선스 키, 티어 관리 등) API 빈약
- **수익 수취**: PayPal 강제 (한국은 Stripe Connect 대상 제외)
- **수수료**: **10% + $0.30/건** (업계 평균 크게 상회)
- **방향성**: 소프트웨어 개발자보다 예술가/교육 콘텐츠 제작자 중심으로 고착화

### 2.3 Stripe (PSP)

- **사업자 없이 가입 불가**: 한국에서 라이브 환경 전환 시 **사업자등록증 필수**. 한국 여신전문금융업법 제약으로 개인 자격 상점 승인 원천 불가
- **개인사업자로 가능**: 글로벌 유저 해외 결제(USD) 수취 완벽 지원
- **수수료**: 3.4% + 300원 (국내), 해외 카드 시 +1~2% 크로스보더 수수료
- **주의**: PSP이므로 글로벌 세금(VAT/Sales Tax) 직접 관리 필요
- **Stripe Atlas 우회**: 미국 법인 원격 설립 가능하나 설립비 $500 + 매년 미국 법인세 신고 의무 → $750 MRR 규모에 부적합

### 2.4 Paddle (MoR)

- **사업자 없이 가입**: 이론상 가능하나, 실질적 컴플라이언스 심사가 매우 보수적
- **심사**: 약관, 환불 정책, 도메인 소유권 등 엄격히 심사. 사업자등록증 없는 1인 프로젝트는 반려율 높음
- **수수료**: 5% + $0.50/건
- **특성**: 안정된 트래픽과 법적 주체를 지닌 스케일업 단계 기업 우대

### 2.5 PayPal (PSP)

- **사업자 없이 가능**: 비즈니스 계정 개인 자격 등록 가능. 단, 거래량 증가 시 사업자등록증 소명 요구 리스크
- **한국 제한**: 한국 내 거주자 간 PayPal 결제 차단 (해외 외화 거래만 가능)
- **치명적 문제 - 환전 수수료**: 기본 4.4% + $0.30 + 원화 강제 환전 시 3~4% 환전 스프레드 → **실효 총 수수료 8%+**
- **세금 대행 안 함** (PSP)

### 2.6 기타 대안

- **Buy Me a Coffee / Ko-fi**: 사업자 불필요, 수수료 0~5%. 그러나 Stripe/PayPal 연결 필수 (한국은 PayPal만 가능 → 고비용 환전). SaaS Webhook/API 지원 극히 빈약
- **Wise**: 매우 낮은 수수료 글로벌 다통화 계좌. 그러나 결제 게이트웨이(결제창) 미제공 → 자동 구독 결제 모델에 부적합

## 5. 한국 세법 및 사업자 등록 규제

### 5.1 사업자 등록 없이 해외 수익 수취

- **합법 여부**: 원천적 불법이 아님. '기타소득' 또는 '사업소득(인적용역)'으로 분류 가능
- **사업자 등록 의무**: 부가가치세법 제8조에 따라 '계속적이고 반복적으로' 용역을 공급하는 자는 사업 개시 20일 이내 등록 의무. SaaS 월 구독은 이 요건에 부합
- **미등록 가산세**: 매출액의 1%
- **외화 수취**: 연간 $50,000 이하 → 자본거래 신고/한국은행 사전 인가 면제. 연 $9,000은 한도의 20% 미만으로 규제 마찰 제로
- **종합소득세**: 이듬해 5월 근로소득(직장 연봉)과 합산 신고 필수. 누락 시 가산세

### 5.2 개인사업자 등록 시

- **등록 절차**: 홈택스에서 공인인증서 기반 실시간 무료 처리
- **업종 코드**: 응용 소프트웨어 개발 및 공급업 (722005) 또는 컴퓨터 프로그래밍 (722000)
- **부가가치세**: 해외 유저 대상 SaaS = '용역의 국외 공급' → **영세율(VAT 0%)** 적용
  - 해외 유저에게 VAT 징수 의무 소멸
  - 국내 매입세액(서버비, 소프트웨어 구독료 등) **전액 환급** 가능
  - 필수 서류: 외화입금증명서 + 영세율 매출명세서 (매년 1월/7월)
  - **일반과세자**로 등록 권장 (간이과세자 아닌)
- **세액 감면**: 청년창업중소기업 세액감면, 중소기업 특별세액감면 대상 가능

### 5.3 직장인 겸업 및 건보료

- **겸업 금지 조항**: 헌법상 직업 선택 자유로 법적 처벌 불가. 단, 회사 인사상 불이익 가능
- **회사 통보 경로**: 유일한 공식 루트 = 국민건강보험료 변동 통지
- **추가 부과 기준**: 보수외소득 연간 **2,000만원 초과** 시 소득월액보험료 부과
- **$750 MRR 시나리오**: 연 매출 약 1,200만원 → 필요경비 차감 후 순수익 1,000만원 미만 → **2,000만원 미달 → 회사 통보 안 됨**

### 5.4 등록 시점의 유연성

- 세법상 '사업 개시 후 20일 이내' 규정은 무형의 SaaS MVP 단계에서 적용 시점이 모호
- **실무적 해석**: 고정적인 클라우드 유지 비용을 초과하는 잉여 현금 흐름이 매월 안정적으로 창출되는 시점 = 실질적 사업 개시일
- LemonSqueezy는 미등록 개인의 테스트 결제를 허용하며, 추후 법인격 정보 업데이트 가능
- Stripe는 1일차부터 사업자 정보 입력 강제 → 유예 기간 확보 불가

## 7. 조건별 시나리오

### 7.1 MVP 단계 (사업자 미등록)

- **플랫폼**: LemonSqueezy (여권 KYC)
- **수취 계좌**: 개인 명의 외화 보통예금 계좌
- **전략**: 행정비용 최소화, 자원을 기능 개발과 마케팅에 집중
- **Gumroad 비추**: 수수료 높고 SaaS API 빈약

### 7.2 스케일업 단계 (개인사업자 전환)

- **시점**: MRR $300~500 안정적 달성 시
- **등록**: 홈택스에서 무료 즉시 발급 (업종코드 722005)
- **선택지**:
  - LemonSqueezy 잔류 (수수료 5% 유지, 세무 편의) + 영세율 매입세액 환급
  - Stripe 전환 (수수료 3.4%로 절감, 세무 직접 관리)
- **대부분의 1인 개발자**: MoR 잔류 + 영세율 환급 하이브리드 전략 채택

## 8. 한국 개인 개발자 실제 사례

### 사례 A: 단계적 확장 모델 (노션 기반 SaaS 개발자 'T')
- Gumroad로 시작 → 높은 환전 비용으로 LemonSqueezy 이관 → 수익 안정화 후 개인사업자 등록
- 건보료 2,000만원 마지노선 모니터링하며 직장과 병행

### 사례 B: 초기 B2B 최적화 모델 (소셜 미디어 분석 앱 'D')
- 론칭 첫날부터 개인사업자 등록 → Stripe Korea 심사 통과
- Stripe Tax + 외화입금증명서 기반 영세율 → 서버비 10% 환급으로 캐시플로우 최적화

### 사례 C: 엄격한 심사 직면 모델 (디자인 에셋 구독 'J')
- Paddle 선택 → 약관 미비/환불 정책 불명확으로 수차례 반려
- 영문 법률 약관 보완 + 사업자등록증 영문 번역 제출 후 최종 승인
- Paddle은 초기 개인 프로젝트에 보수적 심사 적용
